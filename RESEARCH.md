## 1️⃣ 土地使用分區自動判斷

### 🔍 發現

#### 中央政府 WMS 服務
**內政部國土測繪中心**提供全國統一的 WMS 服務：
```
服務名稱：MOI_WMS_005 - 都市計畫土地使用分區
網址：https://cop.land.moi.gov.tw/Services/
```

**技術方式**：
```
1. 地址 → 座標（已完成）
2. 座標 → WMS GetFeatureInfo 查詢
3. 取得使用分區代碼
4. 解析為「住宅區」、「商業區」等
```

---

### ⚠️ 限制與挑戰

#### 問題 1：資料格式複雜
- WMS 回傳 XML/GML 格式
- 需要解析 GIS 專用格式
- 使用分區代碼不統一（各縣市編碼不同）

#### 問題 2：座標系統轉換
- WMS 使用 TWD97（台灣坐標系統）
- Google Maps 使用 WGS84
- 需要座標轉換

#### 問題 3：查詢精度
- WMS 可能回傳多筆資料
- 需判斷最接近的分區
- 邊界地區可能不準確

---

### 💡 實作方案

#### 方案 A：完整自動化（困難度⭐⭐⭐⭐⭐）
**優點**：
- ✅ 全自動，無需人工

**缺點**：
- ❌ 需要 2-3 天開發時間
- ❌ 需要處理複雜的 GIS 格式
- ❌ 準確度可能只有 70-80%
- ❌ 維護成本高（政府API可能變更）

**預估工作量**：
- GIS 格式解析：8 小時
- 座標轉換：4 小時
- 分區代碼對照表：4 小時
- 測試與除錯：8 小時
- **總計：24 小時（3 個工作天）**

## 2️⃣ 建物年份自動查詢

### 🔍 發現

#### 內政部實價登錄開放資料
**資料來源**：
```
內政部不動產交易實價查詢服務網
開放資料下載：https://plvr.land.moi.gov.tw/DownloadOpenData
```

**包含欄位**：
- ✅ 建物完成年月（例：1050101 = 民國105年1月）
- ✅ 建物型態（住宅大樓、透天、華廈等）
- ✅ 總樓層數
- ✅ 交易門牌

---

### ✅ 實作方案

#### 步驟：
```javascript
// 1. 地址正規化
address = "台中市西屯區文華路138巷6號"
normalized = "台中市西屯區文華路138巷"  // 移除樓層

// 2. 查詢實價登錄資料（by 門牌）
data = searchRealPrice(normalized)

// 3. 取得建物年份
if (data.found) {
  year = data.building_year  // 例：105 (民國)
  age = 2026 - 1911 - year   // 轉換為屋齡
  
  評分：
  - 5年內：10分
  - 5-15年：7分
  - 15年以上：5分
  - 查無資料：5分（預設）
}
```

---

### ⚠️ 限制

#### 問題 1：資料量大
- 實價登錄資料檔案每季約 50-100MB
- 不適合前端直接下載

#### 問題 2：地址比對
- 實價登錄地址可能不精確（例：「台中市西屯區文華路138巷6之1號」）
- 需要模糊比對

#### 問題 3：資料延遲
- 實價登錄每季更新
- 新建物可能查不到

---

### 💡 推薦方案

#### GAS 後端整合（困難度⭐⭐⭐）

**實作方式**：
```javascript
// GAS 定期下載實價登錄資料（每月一次）
function updateRealPriceData() {
  // 下載最新一季資料
  data = downloadFromMOI()
  
  // 儲存到 Google Sheets
  sheet.clearContent()
  sheet.setValues(data)
}

// 查詢建物年份
function getBuildingYear(address) {
  // 在 Google Sheets 中查詢
  result = sheet.query(address)
  
  return {
    building_year: result.year,
    building_type: result.type,
    age: calculateAge(result.year)
  }
}
```

**優點**：
- ✅ 後端處理，不影響前端效能
- ✅ Google Sheets 可快速查詢
- ✅ 每月更新一次即可

**缺點**：
- ❌ 需要定期手動更新（或設定 Time-driven trigger）
- ❌ 資料可能不夠即時（最多延遲 3 個月）

**預估工作量**：
- 資料下載與匯入：4 小時
- 查詢功能開發：3 小時
- 地址比對邏輯：3 小時
- **總計：10 小時（1.5 個工作天）**
